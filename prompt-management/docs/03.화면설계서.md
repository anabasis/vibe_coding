# LLM 프롬프트 관리 서비스 화면 설계서

## 1. 공통 요소 (Common Elements)

### 1.1. 헤더 (Header)

- **목적:** 모든 페이지 상단에 위치하여 핵심 기능으로의 빠른 접근을 제공합니다.
- **구성 요소:**
  - **로고 (좌측):**
    - **기능:** 클릭 시 메인 페이지(`/`)로 이동합니다.
  - **네비게이션 메뉴 (우측):**
    - **상태 1: 로그아웃 상태 (비로그인 사용자)**
      - `[로그인]` 버튼: 클릭 시 로그인 페이지(`/login`)로 이동합니다.
    - **상태 2: 로그인 상태 (관리자)**
      - `[프롬프트 관리]` 링크: 클릭 시 프롬프트 관리 페이지(`/admin/prompts`)로 이동합니다.
      - **프로필 이미지/닉네임:**
        - 클릭 시 아래 항목을 포함하는 드롭다운 메뉴가 나타납니다.
          - `프로필 관리`: 클릭 시 프로필 관리 페이지(`/profile`)로 이동합니다.
          - `로그아웃`: 클릭 시 로그아웃 처리 후 메인 페이지로 이동합니다.

### 1.2. 공통 UI/UX 노트

- **로딩 상태:** 데이터 요청 시, 스켈레톤 UI 또는 로딩 스피너를 표시하여 사용자가 대기 중임을 인지시킵니다.
- **사용자 피드백:** '프롬프트 생성', '프롬프트 수정', '프롬프트 삭제' 등 사용자의 주요 액션 완료 시, 화면 우측 하단에 "완료되었습니다"와 같은 토스트(Toast) 팝업을 1~2초간 노출하여 명확한 피드백을 제공합니다.
- **에러 처리:** API 호출 실패 시 사용자에게 명확한 에러 메시지를 표시합니다.

## 2. 개별 화면 명세

### 화면 ID: HOME-001

- **화면명:** 메인 페이지 (TOP10 프롬프트)
- **경로:** `/`
- **접근 권한:** 모든 사용자 (로그인/비로그인)
- **핵심 목적:** 비로그인 사용자에게 TOP10 프롬프트를 표시하고, 제한된 검색 기능을 제공합니다.
- **화면 구성 요소 및 기능:**

  1. **페이지 타이틀:**
     - "인기 프롬프트 TOP10" 또는 "추천 프롬프트"와 같은 헤드라인 문구

  2. **검색 영역 (비로그인 사용자용):**
     - **검색 입력창:**
       - 플레이스홀더: "TOP10 프롬프트에서 검색..."
       - 검색 범위: 제목, 설명, 태그
       - 검색 결과는 TOP10 프롬프트 내에서만 표시됨
     - **검색 버튼:** 검색 실행

  3. **프롬프트 그리드 (Grid):**
     - TOP10 프롬프트만 표시 (조회수 상위 10개)
     - `PromptCard` 컴포넌트가 격자 형태로 정렬됩니다.
     - **정렬 기준:** `view_count` 내림차순 (기본값)

  4. **`PromptCard` (개별 프롬프트 카드):**
     - **이미지 영역:** 프롬프트의 결과물 예시 이미지를 표시합니다.
     - **정보 영역:**
       - `제목`: 프롬프트의 제목 (두 줄 이상일 경우 말줄임 처리)
       - `설명`: 프롬프트 설명 (1-2줄, 말줄임 처리)
       - `태그`: 태그 목록 (최대 3개 표시)
       - `조회수`: 조회수 표시 (예: "조회 1,234회")
     - **버튼 영역:**
       - `[상세보기]` 버튼: 클릭 시 해당 프롬프트의 상세 페이지(`/prompts/[id]`)로 이동합니다.
       - **주의:** 프롬프트 본문(`prompt_text`)은 비공개

  5. **로그인 사용자 전용 영역 (조건부 표시):**
     - 로그인 상태일 경우, "전체 프롬프트 관리하기" 버튼 표시
     - 클릭 시 `/admin/prompts`로 이동

### 화면 ID: LOGIN-001

- **화면명:** 로그인 페이지
- **경로:** `/login`
- **접근 권한:** 모든 사용자
- **핵심 목적:** 관리자 로그인을 통해 프롬프트 관리 권한을 획득합니다.
- **화면 구성 요소 및 기능:**

  1. **로그인 폼:**
     - **이메일 입력 필드:**
       - 타입: `email`
       - 필수 입력
       - 유효성 검사: 이메일 형식 확인
     - **비밀번호 입력 필드:**
       - 타입: `password`
       - 필수 입력
       - 최소 길이 제한 (예: 8자 이상)
     - **`[로그인]` 버튼:**
       - 클릭 시 인증 처리
       - 로딩 상태 표시
       - 성공 시 메인 페이지 또는 이전 페이지로 리다이렉트
     - **에러 메시지:**
       - 로그인 실패 시 "이메일 또는 비밀번호를 확인해주세요." 메시지 표시

  2. **추가 링크:**
     - "비밀번호를 잊으셨나요?" (향후 구현)

---

### 화면 ID: ADMIN-001

- **화면명:** 프롬프트 관리 대시보드
- **경로:** `/admin/prompts`
- **접근 권한:** 로그인 사용자만 (관리자)
- **핵심 목적:** 모든 프롬프트를 관리(생성/수정/삭제/조회)할 수 있는 관리자 페이지입니다.
- **화면 구성 요소 및 기능:**

  1. **페이지 헤더:**
     - **페이지 타이틀:** "프롬프트 관리"
     - **`[새 프롬프트 생성]` 버튼:**
       - 클릭 시 프롬프트 생성 모달 또는 페이지로 이동

  2. **통계 영역 (선택사항):**
     - 전체 프롬프트 수
     - 최근 7일간 생성된 프롬프트 수
     - 평균 조회수

  3. **검색 및 필터 영역:**
     - **검색 입력창:**
       - 플레이스홀더: "전체 프롬프트에서 검색..."
       - 검색 범위: 제목, 설명, 태그, 프롬프트 본문
       - 실시간 검색 또는 검색 버튼 클릭
     - **필터 옵션:**
       - 공개/비공개 필터
       - 태그 필터 (다중 선택 가능)
     - **정렬 옵션:**
       - 최신순, 조회수순, 제목순

  4. **프롬프트 목록 테이블:**
     - **테이블 컬럼:**
       - `제목`: 프롬프트 제목 (클릭 시 상세 페이지로 이동)
       - `태그`: 태그 목록 (배지 형태)
       - `조회수`: 조회수
       - `생성일`: 생성 날짜
       - `상태`: 공개/비공개
       - `액션`: 수정/삭제 버튼
     - **페이지네이션:**
       - 한 페이지당 20개 또는 50개 표시
       - 이전/다음 페이지 네비게이션

  5. **액션 버튼 (각 행):**
     - **`[수정]` 버튼:**
       - 클릭 시 프롬프트 수정 모달 또는 페이지로 이동
     - **`[삭제]` 버튼:**
       - 클릭 시 삭제 확인 다이얼로그 표시
       - 확인 시 프롬프트 삭제 및 목록 갱신

### 화면 ID: ADMIN-002

- **화면명:** 프롬프트 생성/수정 페이지
- **경로:** 
  - 생성: `/admin/prompts/new`
  - 수정: `/admin/prompts/[id]/edit`
- **접근 권한:** 로그인 사용자만 (관리자)
- **핵심 목적:** 새로운 프롬프트를 생성하거나 기존 프롬프트를 수정합니다.
- **화면 구성 요소 및 기능:**

  1. **페이지 헤더:**
     - 생성 모드: "새 프롬프트 생성"
     - 수정 모드: "프롬프트 수정"

  2. **프롬프트 입력 폼:**
     - **기본 정보:**
       - `제목` (필수): 텍스트 입력
       - `설명` (필수): 텍스트 영역 (여러 줄)
       - `프롬프트 본문` (필수): 텍스트 영역 (여러 줄, 코드 에디터 권장)
     - **이미지:**
       - `이미지 업로드`: 여러 이미지 업로드 가능
       - 업로드된 이미지 미리보기
       - 이미지 삭제 기능
     - **태그:**
       - 태그 입력 (쉼표로 구분 또는 개별 입력)
       - 기존 태그 목록에서 선택 가능
     - **설정:**
       - `공개 여부`: 토글 스위치 (기본값: 공개)
       - `조회수 초기화`: 수정 시 선택 가능 (기본값: 유지)

  3. **액션 버튼:**
     - **`[저장]` 버튼:**
       - 폼 유효성 검사 후 저장
       - 성공 시 프롬프트 관리 페이지로 리다이렉트
     - **`[취소]` 버튼:**
       - 변경사항 저장하지 않고 이전 페이지로 이동
       - 변경사항이 있을 경우 확인 다이얼로그 표시

  4. **미리보기 영역 (선택사항):**
     - 입력한 정보를 카드 형태로 미리보기

### 화면 ID: PROD-001

- **화면명:** 프롬프트 상세 페이지
- **경로:** `/prompts/[id]`
- **접근 권한:** 모든 사용자 (역할별로 다른 정보 표시)
- **핵심 목적:** 프롬프트의 상세 정보를 확인합니다. 사용자 역할에 따라 표시되는 정보가 다릅니다.
- **화면 구성 요소 및 기능:**

  1. **프롬프트 기본 정보:**
     - `제목`: 프롬프트의 전체 제목
     - `설명`: 프롬프트의 상세 설명
     - `태그`: 태그 목록 (배지 형태)
     - `조회수`: 조회수 표시
     - `생성일`: 생성 날짜

  2. **이미지 갤러리 영역:**
     - 결과물 예시 이미지들을 여러 장 보여주는 슬라이드 또는 갤러리 형태
     - 이미지 확대 보기 기능

  3. **프롬프트 본문 영역:**
     - **상태 1: 비로그인 사용자**
       - "로그인 후 프롬프트 본문을 확인할 수 있습니다." 메시지 표시
       - `[로그인하기]` 버튼: 로그인 페이지로 이동
     - **상태 2: 로그인 사용자 (관리자)**
       - 실제 프롬프트 본문(`prompt_text`) 표시
       - 코드 블록 형태로 표시 (구문 강조 권장)
       - `[복사]` 버튼: 클릭 시 프롬프트 본문을 클립보드에 복사
       - `[수정하기]` 버튼: 프롬프트 수정 페이지로 이동

  4. **관리자 전용 액션 영역:**
     - 로그인 상태일 경우에만 표시
     - `[수정]` 버튼: 프롬프트 수정 페이지로 이동
     - `[삭제]` 버튼: 삭제 확인 다이얼로그 표시

  5. **조회수 증가:**
     - 페이지 로드 시 조회수 자동 증가 (중복 증가 방지 필요)

### 화면 ID: USER-001

- **화면명:** 프로필 관리
- **경로:** `/profile`
- **접근 권한:** 로그인 사용자만
- **핵심 목적:** 사용자가 자신의 프로필 정보를 수정합니다.
- **화면 구성 요소 및 기능:**

  1. **페이지 타이틀:** "프로필 관리"

  2. **프로필 이미지 영역:**
     - 현재 프로필 이미지를 원형으로 표시합니다.
     - 이미지 위에 마우스를 올리면 `[변경]` 아이콘이 나타나며, 클릭 시 파일 업로드 창이 열립니다.
     - 이미지 업로드 후 미리보기 표시

  3. **정보 입력 폼:**
     - **이메일:**
       - 사용자의 가입 이메일을 표시합니다.
       - 수정이 불가능한 `disabled` 상태의 입력 필드로 표시합니다.
     - **닉네임:**
       - **읽기 모드 (기본):** 현재 닉네임을 텍스트로 표시하고, 옆에 `[수정]` 버튼을 둡니다.
       - **수정 모드:** `[수정]` 버튼 클릭 시, 텍스트가 `input` 필드로 변경되고 `[저장]`, `[취소]` 버튼이 나타납니다.
         - `[저장]`: 클릭 시 입력된 닉네임으로 정보를 업데이트하고 토스트 알림 후 읽기 모드로 전환합니다.
         - `[취소]`: 클릭 시 변경사항을 저장하지 않고 읽기 모드로 전환합니다.

  4. **저장 버튼:**
     - 프로필 이미지와 닉네임 변경사항을 모두 저장

## 3. 사용자 플로우 (User Flow)

### 3.1. 비로그인 사용자 플로우

1. **메인 페이지 접근** (`/`)
   - TOP10 프롬프트 목록 확인
   - 검색 기능 사용 (TOP10 내에서만)
   - 프롬프트 상세 페이지 접근 (제한된 정보만 표시)

2. **프롬프트 상세 페이지** (`/prompts/[id]`)
   - 기본 정보 및 이미지 확인
   - 프롬프트 본문은 비공개 (로그인 유도 메시지)

### 3.2. 로그인 사용자 (관리자) 플로우

1. **로그인** (`/login`)
   - 이메일/비밀번호 입력
   - 인증 성공 시 메인 페이지 또는 관리 페이지로 이동

2. **프롬프트 관리** (`/admin/prompts`)
   - 전체 프롬프트 목록 확인
   - 검색 및 필터링
   - 프롬프트 생성/수정/삭제

3. **프롬프트 생성** (`/admin/prompts/new`)
   - 프롬프트 정보 입력
   - 이미지 업로드
   - 태그 설정
   - 저장 후 관리 페이지로 이동

4. **프롬프트 수정** (`/admin/prompts/[id]/edit`)
   - 기존 정보 로드
   - 정보 수정
   - 저장 후 관리 페이지로 이동

5. **프롬프트 상세** (`/prompts/[id]`)
   - 전체 정보 확인 (본문 포함)
   - 수정/삭제 액션 수행

## 4. 반응형 디자인 고려사항

### 4.1. 모바일 (320px ~ 768px)
- 헤더: 햄버거 메뉴로 변경
- 프롬프트 그리드: 1열 또는 2열
- 테이블: 카드 형태로 변환
- 검색 영역: 전체 너비 사용

### 4.2. 태블릿 (768px ~ 1024px)
- 프롬프트 그리드: 2열 또는 3열
- 테이블: 스크롤 가능한 테이블

### 4.3. 데스크톱 (1024px 이상)
